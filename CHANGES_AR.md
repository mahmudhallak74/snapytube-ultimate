# 🛠️ شو تعدّل بالمشروع

## 🔒 ثغرات وأخطاء تصليحها

1. **ثغرة تسريب ملفات (`/api/video/<filename>`)**
   كان في مطابقة جزئية (`if filename in f`) — يعني حدا يقدر يبعت اسم قصير ويجيب ملف غلط من مجلد غيره، أو يجرب `../` (path traversal). صار فيه:
   - تحقق `os.path.realpath` إن المسار فعلياً داخل مجلد المستخدم فقط.
   - مطابقة اسم دقيقة (exact match) بدل الجزئية.

2. **ما في حماية من إغراق السيرفر بطلبات (Rate limiting)**
   ضفت حد أقصى (12 طلب تحميل/دقيقة لكل IP) عشان ما حدا يعمل سبام ع السيرفر لو الرابط العالمي مفتوح.

3. **حجم الطلبات غير محدود** → ضفت `MAX_CONTENT_LENGTH` (1MB) يمنع أي حدا يبعت جسم طلب ضخم عشوائي.

4. **أسماء ملفات غير آمنة** → فعّلت `restrictfilenames` بـ yt-dlp، يمنع مشاكل الأحرف الخاصة/الرموز اللي ممكن تكسر المسار على أندرويد.

5. **ما في حد أقصى فعلي لحجم الفيديو** رغم إنه معرّف بالـ Config → ربطته فعلياً بـ `max_filesize` بخيارات yt-dlp.

6. **أداء**: كان بيعمل `setup_for_client` (إنشاء مجلدات) على *كل* طلب حتى للملفات الثابتة و`/api/health` → صار يتخطاها.

7. **رسائل الأخطاء** كانت بترجع `str(e)` الخام للمستخدم بمسارات كاملة أحياناً → صارت رسائل عامة آمنة.

## ⚡ تحديث تلقائي

- كل ما يفتح السيرفر (يعني كل تشغيل لـ `start.sh`) فيه ثريد بالخلفية يحدّث `yt-dlp` تلقائياً بدون ما يوقف تشغيل السيرفر.
- الواجهة بتستدعي `/api/check-update` أول ما تفتح الصفحة وبتطلع إشعار أخضر لو صار تحديث.

## 🖥️ الواجهة الرئيسية

ضفت زرين بالأعلى:
- **تحميل بالرابط** → نفس صفحة التحميل القديمة.
- **الملفات المحفوظة** → صفحة جديدة فيها:
  - مكان الحفظ الفعلي على الجهاز (المسار).
  - تبويبات: فيديو / صور / صوت (عن طريق `/api/files` الجديد اللي بيفحص كل الامتدادات).

## 📱 تجميعه كتطبيق أندرويد (WebView + apktool)

المشروع أصلاً سيرفر Flask بيشتغل محلياً على الهاتف عن طريق Termux (`localhost:5001`)، فـ"التطبيق" الأسهل هو WebView بسيط بيفتح `http://127.0.0.1:5001` — وهاد هو نفس الأسلوب يلي حكينا عنه قبل:

1. خد أي base APK فيه WebView بسيط (فاضي)، فكّه بـ:
   ```
   apktool d base.apk -o SnapYTube_APK
   ```
2. بدّل الرابط اللي بيفتحه الـ WebView جوا الكود smali (أو `MainActivity`) لـ:
   `http://127.0.0.1:5001`
3. تأكد إن صلاحية `<uses-permission android:name="android.permission.INTERNET"/>` موجودة بـ `AndroidManifest.xml`.
4. أعد تجميعه:
   ```
   apktool b SnapYTube_APK -o SnapYTube_unsigned.apk
   ```
5. وقّعه بمفتاحك متل ما بتعمل عادة (`apksigner` أو `jarsigner` + `zipalign`).
6. لازم سيرفر Flask (`bash start.sh`) يكون شغّال بالخلفية عن طريق Termux:Boot قبل ما تفتح التطبيق، لأنه التطبيق مجرد واجهة WebView بتعرض نفس الصفحة.

> ⚠️ ملاحظة: هاد أسلوب WebView بسيط بيعتمد إنه Termux شغّال بالخلفية. إذا بدك تطبيق مستقل بالكامل بدون Termux (يعني الداونلود إنجن جوا الـ APK نفسه)، هاد مشروع تاني بالكامل (لازم Chaquopy أو Kivy لتضمين بايثون جوا أندرويد) — قولّي إذا هاد يلي بدك ياه وبفصّلها.
